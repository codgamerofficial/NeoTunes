# Payment Gateways Integration Plan

## Overview
Integrate Cashfree, PayPal, and Wise payment gateways into the existing React/Vite application while maintaining backward compatibility with Razorpay.

## Current State
- Existing Razorpay integration in `src/lib/payments.ts` and `src/modules/Billing.tsx`
- Supabase backend for user data and authentication
- Vite-based React 19 application with TypeScript
- Tailwind CSS 4 design system with Framer Motion animations

## Implementation Plan

### 1. Cashfree Integration
#### Files to Modify:
- `src/lib/payments.ts` - Add Cashfree client and helper functions
- `src/modules/Billing.tsx` - Add Cashfree payment option UI
- `src/components/PaymentMethods.tsx` (new) - Unified payment method selector

#### Implementation Details:
- Add Cashfree React SDK and Cashfree JS dependencies
- Create server endpoint for payment session generation (`/api/payments/cashfree/session`)
- Implement Cashfree checkout component with card fields
- Add webhook verification endpoint (`/api/webhooks/cashfree`)
- Handle success/failure callbacks and redirect to order confirmation

### 2. PayPal Integration
#### Files to Modify:
- `src/lib/payments.ts` - Add PayPal client functions
- `src/modules/Billing.tsx` - Add PayPal payment option
- `src/components/PayPalButton.tsx` (new) - React PayPal button component

#### Implementation Details:
- Add `@paypal/react-paypal-js` dependency
- Create PayPal order creation endpoint (`/api/paypal/create-order`)
- Implement PayPalButtons component with proper styling
- Add payment verification endpoint (`/api/paypal/verify-payment`)
- Handle both sandbox and production modes

### 3. Wise Integration
#### Files to Modify:
- `src/lib/payments.ts` - Add Wise API client
- `src/modules/Profile.tsx` (seller section) - Add payout management
- `src/lib/wise.ts` (new) - Wise API service wrapper

#### Implementation Details:
- Implement OAuth 2.0 client credentials flow for authentication
- Create quote generation endpoint (`/api/wise/quote`)
- Implement recipient account management
- Add transfer creation endpoint (`/api/wise/transfer`)
- Add webhook handler for transfer status updates (`/api/webhooks/wise`)

### 4. Unified Payment Interface
#### Files to Create/Modify:
- `src/lib/paymentFacade.ts` (new) - Unified payment interface
- `src/components/PaymentMethodSelector.tsx` (new) - UI for selecting payment method
- `src/modules/Billing.tsx` - Integrate unified payment system

#### Implementation Details:
- Create abstraction layer that selects appropriate gateway based on:
  - User location (IN for Razorpay/Cashfree, International for PayPal/Wise)
  - Transaction amount thresholds
  - Payment method preferences
- Maintain consistent API across all gateways
- Handle currency conversion and fees transparently
- Provide fallback mechanisms for failed gateway attempts

### 5. Database Updates
#### Supabase Schema Changes:
- Add `payment_methods` table to store user preferences
- Add `payment_transactions` table with gateway-specific fields
- Add `payout_accounts` table for Wise recipient details
- Add indexes for efficient querying

### 6. Security Considerations
- Never expose secret keys on frontend
- Implement proper webhook signature verification
- Use environment variables for all API credentials
- Implement rate limiting on payment endpoints
- Log all payment attempts for audit trail
- Ensure PCI DSS compliance for card data handling

### 7. Testing Strategy
- Unit tests for payment facade logic
- Integration tests for each gateway flow
- End-to-end tests for checkout process
- Manual testing in sandbox environments
- Load testing for peak traffic scenarios

### 8. Rollback Plan
- Feature flags for each new payment gateway
- Database migrations are backward compatible
- Ability to disable individual gateways without affecting others
- Monitoring and alerting for payment failure rates

## Estimated Effort
- Cashfree Integration: 2 days
- PayPal Integration: 1.5 days  
- Wise Integration: 2 days
- Unified Interface: 1 day
- Testing & Documentation: 1.5 days
- Total: 8 days

## Dependencies
- `@cashfreepayments/pg-react` and `@cashfreepayments/cashfree-js`
- `@paypal/react-paypal-js`
- `axios` for Wise API calls
- Existing dependencies: react, framer-motion, supabase

## Risks and Mitigations
1. **Webhook Security**: Implement strict signature verification and IP whitelisting
2. **Payment Failures**: Add retry logic and fallback to alternative gateways
3. **API Changes**: Abstract gateway specifics behind facade interface
4. **Compliance**: Use PCI-compliant hosted fields where available
5. **User Experience**: Maintain consistent UI across all payment methods

## Success Criteria
- All payment gateways functional in test mode
- Unified payment interface working seamlessly
- Proper error handling and user feedback
- Webhook signatures verified correctly
- No regression in existing Razorpay functionality
- Analytics tracking for payment method usage
